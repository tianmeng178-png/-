## 系统要求

### 最低配置
- **操作系统**: Windows 10/11, macOS 12+, Ubuntu 22.04+
- **CPU**: 4核及以上
- **内存**: 8GB RAM (推荐16GB+)
- **磁盘空间**: 20GB 可用空间

### 软件依赖
- **Python**: 3.10+
- **Node.js**: 18+
- **npm**: 9+
- **Docker**: 20.10+ (可选，用于Docker环境)
- **Docker Compose**: 2.0+ (可选)

## 本地开发环境搭建

### 1. 克隆项目
```bash
git clone <repository-url>
cd heat_exchanger_ai
```

### 2. 后端环境配置

#### 2.1 创建Python虚拟环境
```bash
# Windows
python -m venv venv
venv\Scripts\activate

# macOS/Linux
python3 -m venv venv
source venv/bin/activate
```

#### 2.2 安装Python依赖
```bash
cd backend
pip install -r requirements.txt
```

#### 2.3 启动后端服务
```bash
python main.py
```
后端服务将在 `http://localhost:8000` 启动

### 3. 前端环境配置

#### 3.1 安装Node.js依赖
```bash
cd frontend
npm install
```

#### 3.2 启动前端开发服务器
```bash
npm run dev
```
前端服务将在 `http://localhost:3000` 启动

### 4. 验证安装

#### 4.1 健康检查
访问 `http://localhost:8000/api/health` 验证后端健康状态

#### 4.2 访问前端
打开浏览器访问 `http://localhost:3000`

## Docker开发环境

### 1. 使用Docker Compose启动
```bash
# 构建并启动所有服务
docker-compose up --build

# 后台运行
docker-compose up -d --build
```

### 2. 查看服务状态
```bash
docker-compose ps
```

### 3. 查看日志
```bash
# 查看所有服务日志
docker-compose logs -f

# 查看特定服务日志
docker-compose logs -f backend
docker-compose logs -f frontend
```

### 4. 停止服务
```bash
docker-compose down

# 停止并删除数据卷
docker-compose down -v
```

## 生产环境部署

### 前端部署 (Vercel)

#### 1. 准备部署
```bash
cd frontend
npm run build
```

#### 2. 配置Vercel
项目已包含 `vercel.json` 配置文件，直接连接GitHub仓库即可自动部署。

### 后端部署 (云服务器)

#### 1. 环境变量配置
创建 `.env` 文件：
```env
# API配置
API_HOST=0.0.0.0
API_PORT=8000

# CORS配置
ALLOWED_ORIGINS=https://your-domain.com

# LLM配置（可选）
LLM_API_KEY=your-api-key
LLM_MODEL=deepseek-chat
```

#### 2. 使用Gunicorn部署
```bash
pip install gunicorn
gunicorn -w 4 -k uvicorn.workers.UvicornWorker main:app
```

#### 3. 使用Systemd管理服务
创建 `/etc/systemd/system/heat-exchanger.service`：
```ini
[Unit]
Description=Heat Exchanger AI Backend
After=network.target

[Service]
Type=notify
User=www-data
WorkingDirectory=/path/to/backend
Environment="PATH=/path/to/venv/bin"
ExecStart=/path/to/venv/bin/gunicorn -w 4 -k uvicorn.workers.UvicornWorker main:app
Restart=always

[Install]
WantedBy=multi-user.target
```

启动服务：
```bash
sudo systemctl daemon-reload
sudo systemctl enable heat-exchanger
sudo systemctl start heat-exchanger
```

## OpenFOAM环境配置

### Windows (WSL2 + Ubuntu)

#### 1. 启用WSL2
```powershell
# 以管理员身份运行PowerShell
wsl --install
```

#### 2. 安装Ubuntu 22.04
从Microsoft Store安装Ubuntu 22.04 LTS

#### 3. 在WSL2中安装OpenFOAM v11
```bash
# 进入WSL2
wsl

# 更新包管理器
sudo apt update
sudo apt upgrade -y

# 安装OpenFOAM依赖
sudo apt install -y build-essential flex bison cmake zlib1g-dev libopenmpi-dev openmpi-bin

# 添加OpenFOAM仓库
curl https://dl.openfoam.com/add-debian-repo.sh | sudo bash

# 安装OpenFOAM v11
sudo apt install openfoam11

# 配置环境变量
echo "source /usr/lib/openfoam/openfoam11/etc/bashrc" >> ~/.bashrc
source ~/.bashrc

# 验证安装
foamVersion
```

### Linux (原生Ubuntu)

```bash
# 更新系统
sudo apt update
sudo apt upgrade -y

# 安装OpenFOAM v11
curl https://dl.openfoam.com/add-debian-repo.sh | sudo bash
sudo apt install openfoam11

# 配置环境
echo "source /usr/lib/openfoam/openfoam11/etc/bashrc" >> ~/.bashrc
source ~/.bashrc

# 验证
foamVersion
```

### macOS

目前OpenFOAM在macOS上支持有限，推荐使用Docker或WSL2。

## 常见问题

### Q: 后端启动失败，端口被占用？
A: 修改 `backend/main.py` 中的端口号，或停止占用端口的进程：
```bash
# Windows
netstat -ano | findstr :8000
taskkill /F /PID <process-id>

# Linux/macOS
lsof -ti :8000 | xargs kill -9
```

### Q: 前端无法连接后端？
A: 检查 `frontend/src/services/api.ts` 中的API地址配置，确保与后端地址一致。

### Q: Docker容器无法启动？
A: 检查Docker是否正常运行，查看日志：
```bash
docker-compose logs
```
